# Does your AI agent trust any file?

The file system is a formidable attack surface for any AI agent that is allowed to handle data and execute commands.

Every file an agent touches can become an attack vector for hidden prompt injection: polyglot files that look like one thing but execute as another, disguised executables, and crafted archives designed to exploit parser vulnerabilities.

Correct file type identification is the first defence gate. If this control is missing or unreliable, the agent will end up making decisions based on vibes and file extensions.

Traditional methods, such as the UNIX file command, pattern-based recognition, or extension-based heuristics, always leave a gap — and that is where attacks happen.

Google considers this unacceptable for large-scale AI agent usage, and proposes Magika: an internal project now released as open source on GitHub.

Magika is a tool as efficient as file, but powered by machine learning and Google’s operational experience. It has an average inference time of around 5 milliseconds per file, and it is hard to fool because its small 3.1 MB model is designed to do one thing only: identify file types.

Magika is limited to 214 known file types and cannot be expanded without retraining the model. That is a limitation, but it also reduces the risk of malicious tampering.

Ba aware: Magika is not a malware detector, not a threat detector, and not a large language model. Your agentic workflow needs much more for secure operations.
